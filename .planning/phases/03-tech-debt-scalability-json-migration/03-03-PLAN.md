**File 1: `src/audit/builtin/n_plus_one_queries.json`**
Per D-05, single cross-language file using match_pattern. Per D-06, broadest structural pattern: call expression inside loop body. No `"languages"` field (cross-language). Document dropped DB-name filters in description per D-05.

The match_pattern S-expression uses alternation `[...]` for multiple loop kinds. The query targets TypeScript/JavaScript grammar node kinds (for_statement, for_in_statement, while_statement, do_statement). Languages whose grammars use different node kinds (e.g., Rust's for_expression, Python's for_statement but different nesting) will have the query silently rejected by execute_match_pattern -- accepted limitation per D-06.

Use only ONE capture to avoid duplicate findings (Pitfall 4 from RESEARCH.md).

```json
{
  "pipeline": "n_plus_one_queries",
  "category": "scalability",
  "description": "Detects call expressions inside loop bodies (potential N+1 query pattern). This JSON version uses structural pattern matching only. It does NOT filter by DB/ORM/HTTP method names as the Rust version did (findOne, findUnique, findMany, find, findById, query, execute, axios.get/post/put/delete/patch, http.get/post/request, fetch, request, Model.find, db.collection). Expect higher false positive rate.",
  "graph": [
    {
      "match_pattern": "[(for_statement body: (_ (expression_statement (call_expression) @call))) (for_in_statement body: (_ (expression_statement (call_expression) @call))) (while_statement body: (_ (expression_statement (call_expression) @call))) (do_statement body: (_ (expression_statement (call_expression) @call)))]"
    },
    {
      "flag": {
        "pattern": "query_in_loop",
        "message": "Call expression inside loop body -- potential N+1 query pattern",
        "severity": "warning"
      }
    }
  ]
}
```
