pub fn resolve_severity(&self, node: &PipelineNode) -> String {
    if let Some(ref map) = self.severity_map {
        for entry in map {
            let matches = match &entry.when {
                None => true, // no condition = default
                Some(wc) => wc.is_empty() || wc.eval_metrics(node),
            };
            if matches {
                return entry.severity.clone();
            }
        }
    }
    self.severity.clone().unwrap_or_else(|| "warning".to_string())
}

**Problem:** When a JSON pipeline's flag config has `severity_map` with only conditional `when` entries (no catch-all entry without `when`) AND no bare `"severity"` field, `resolve_severity` currently falls back to `"warning"` for ALL nodes that don't match any when clause. This means below-threshold functions (e.g., CC=3) get flagged as warnings. The metric pipelines (cyclomatic_complexity, function_length, cognitive_complexity, comment_to_code_ratio) rely on suppression -- nodes below threshold should produce NO finding.

**Fix Part 1 -- src/graph/pipeline.rs:**

Change `resolve_severity` return type from `String` to `Option<String>`. The logic change is:
- If severity_map exists and a `when` clause matches, return `Some(entry.severity.clone())`
- If severity_map exists, no `when` clause matches, AND `self.severity` is `Some(s)`, return `Some(s)` (existing bare severity fallback)
- If severity_map exists, no `when` clause matches, AND `self.severity` is `None`, return `None` (SUPPRESS -- this is the new behavior)
- If severity_map is None and `self.severity` is `Some(s)`, return `Some(s)`
- If severity_map is None and `self.severity` is None, return `Some("warning".to_string())` (backward compat for no-config flags)

Updated implementation (replace lines 433-446):
```rust
    /// Resolve the effective severity given the current node's metrics.
    /// Checks severity_map entries in order; first matching `when` wins.
    /// Falls back to `severity` field if no severity_map entry matches.
    /// Returns None (suppresses finding) when severity_map has only conditional
    /// entries, none match, and no bare `severity` field exists.
    pub fn resolve_severity(&self, node: &PipelineNode) -> Option<String> {
        if let Some(ref map) = self.severity_map {
            for entry in map {
                let matches = match &entry.when {
                    None => true, // no condition = default
                    Some(wc) => wc.is_empty() || wc.eval_metrics(node),
                };
                if matches {
                    return Some(entry.severity.clone());
                }
            }
            // severity_map existed but no entry matched.
            // If there's a bare severity field, use it as fallback.
            // If not, suppress the finding (return None).
            self.severity.clone().map(Some).unwrap_or(None)
        } else {
            // No severity_map at all -- use severity field or default "warning"
            Some(self.severity.clone().unwrap_or_else(|| "warning".to_string()))
        }
    }
```
