3. **path_traversal_python.json** -- pipeline: "path_traversal". Rust detects os.path.join with dynamic args. Simplified: match attribute calls to join on os.path. Pattern: "unvalidated_path_join", severity: "warning".

4. **insecure_deserialization_python.json** -- pipeline: "insecure_deserialization". Rust detects pickle.loads/yaml.load without Loader, marshal.loads. Simplified: match attribute calls to loads on pickle/marshal, load on yaml. Pattern: "insecure_deserialization", severity: "error".

5. **xxe_format_string_python.json** -- pipeline: "xxe_format_string". Rust detects both XXE (lxml etree parse) and format string (%s in SQL). Read Rust source for patterns. Pattern: as per Rust fn. Severity: "warning".

The detection logic uses has_nested_quantifier() which is NOT expressible in match_pattern (it's a string analysis function). Document: "Simplified from Rust: flags all re.compile/re.search/re.match calls. Cannot verify nested quantifier presence in match_pattern."

Simplified per D-09: match open() calls, append()/extend()/add() in for_statement bodies, `__del__` function definitions. Pattern: "potential_memory_leak", severity: "warning".
