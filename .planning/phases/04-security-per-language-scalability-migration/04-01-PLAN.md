For each pipeline, read the Rust source file first, extract the tree-sitter query patterns, and translate to match_pattern S-expressions:

1. **integer_overflow_rust.json** -- pipeline: "integer_overflow", category: "security". Rust uses multi-step: finds functions with params, then binary_expression with * or +. Simplified: match `(binary_expression operator: ["*" "+"])` broadly. Pattern: "unchecked_arithmetic", severity: "warning", message: "Arithmetic operation without overflow check -- verify bounds on untrusted input".

2. **unsafe_memory_rust.json** -- pipeline: "unsafe_memory", category: "security". Rust detects transmute, pointer arithmetic (.offset/.add/.sub), raw deref in unsafe blocks. Simplified: match `(unsafe_block) @unsafe_blk` broadly (cannot filter sub-operations in single match_pattern). Pattern: "unsafe_memory_operation", severity: "warning", message: "Unsafe block detected -- verify no transmute, pointer arithmetic, or raw dereference of untrusted data". Document in description: "Simplified from Rust: flags all unsafe blocks rather than specific sub-operations (transmute, pointer arithmetic, raw deref). Precision reduced."

3. **race_conditions_rust.json** -- pipeline: "race_conditions", category: "security". Pure tree-sitter: `(static_item (mutable_specifier) @mut) @item`. Pattern: "static_mut", severity: "error", message: "static mut variable creates data race potential". CRITICAL: must include "languages": ["rust"] to avoid conflict with Go's race_conditions (same pipeline name).

4. **path_traversal_rust.json** -- pipeline: "path_traversal", category: "security". Rust uses multi-step param extraction + path method cross-reference. Simplified: match method calls .join() and .push() on any receiver. Use `(call_expression function: (field_expression field: (field_identifier) @method) @call)` and document precision loss. Pattern: "unvalidated_path_operation", severity: "warning". Document: "Simplified from Rust: flags all .join()/.push() method calls, not just those with param-derived arguments."

5. **resource_exhaustion_rust.json** -- pipeline: "resource_exhaustion", category: "security". Rust finds with_capacity scoped calls + .reserve() method calls inside parameterized functions. Simplified: match scoped calls containing "with_capacity" and .reserve() calls. Use `(call_expression function: (scoped_identifier) @fn)` broadly. Pattern: "unbounded_allocation", severity: "warning". Document precision loss.

6. **panic_dos_rust.json** -- pipeline: "panic_dos", category: "security". Rust finds .unwrap()/.expect(), index expressions, narrowing casts in parameterized functions. Simplified: match .unwrap() and .expect() method calls. Use `(call_expression function: (field_expression field: (field_identifier) @method) @call)`. Pattern: "unwrap_untrusted", severity: "warning". Document: "Simplified from Rust: flags all unwrap/expect calls, not just in parameterized functions. Index expressions and narrowing casts omitted."

7. **type_confusion_rust.json** -- pipeline: "type_confusion", category: "security". Rust finds transmute calls + union definitions. Simplified: match union_item definitions. Use `(union_item name: (type_identifier) @name) @union`. Pattern: "union_type_confusion", severity: "warning". Document: "Simplified from Rust: flags union definitions. Transmute detection overlaps with unsafe_memory pipeline."

8. **toctou_rust.json** -- pipeline: "toctou", category: "security". Rust finds if_expression with path check in condition + file op in body (text containment). Simplified: match method calls to .exists(), .is_file(), .is_dir(). Use `(call_expression function: (field_expression field: (field_identifier) @method) @call)`. Pattern: "path_check_use_race", severity: "warning". Document: "Simplified from Rust: flags all .exists()/.is_file()/.is_dir() calls, not just those in if conditions followed by file operations."

9. **memory_leak_indicators_rust.json** -- pipeline: "memory_leak_indicators", category: "scalability". Rust detects Box::leak, mem::forget, ManuallyDrop::new (scoped calls) + push/insert/extend in loops. Use `(call_expression function: (scoped_identifier) @fn)` for scoped calls broadly. Pattern: "potential_memory_leak", severity: "warning". Document: "Simplified from Rust: flags all scoped identifier calls. Cannot filter to only Box::leak/mem::forget/ManuallyDrop::new names via match_pattern (predicate support uncertain). Also does not detect loop-body growth patterns separately."

After creating all 9 files, run `cargo test` to verify JSON parses correctly and engine loads them.
