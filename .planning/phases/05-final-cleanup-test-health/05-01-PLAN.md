**Per-pipeline JSON creation instructions:**

1. **panic_detection_rust.json** -- `"pipeline": "panic_detection"`. Use match_pattern for method calls (.unwrap(), .expect()) + macro invocations (panic!, todo!, unimplemented!, unreachable!). Two-query approach: use `[(call_expression function: (field_expression field: (field_identifier) @method)) (macro_invocation macro: (identifier) @macro_name)]` to catch both. Pattern: "panic_prone_call", severity: "warning". Description: document that .lock().unwrap() filtering and severity graduation are simplified.

2. **clone_detection_rust.json** -- `"pipeline": "clone_detection"`. Match .clone() method calls: `(call_expression function: (field_expression field: (field_identifier) @method))` and document simplified from Rust: flags all .clone() calls, not just those on large types or in hot paths. Pattern: "unnecessary_clone", severity: "info".

3. **god_object_detection_rust.json** -- `"pipeline": "god_object_detection"`. Use `select: "symbol"` with `where: {kind: ["struct", "class"]}`, `group_by: "file"`, `count: {threshold: {gte: 10}}` to detect large types. Pattern: "god_object", severity: "warning". Document: struct field count not available in JSON DSL; uses symbol count per file as proxy.

4. **stringly_typed_rust.json** -- `"pipeline": "stringly_typed"`. Match function parameters with string type: `(parameter (reference_type (primitive_type) @ty))` to catch `&str` params. Pattern: "stringly_typed_api", severity: "info". Document: simplified, flags &str params broadly.

5. **must_use_ignored_rust.json** -- `"pipeline": "must_use_ignored"`. Match expression statements with function calls (return value discarded): `(expression_statement (call_expression) @call)`. Pattern: "must_use_ignored", severity: "info". Document: cannot check for #[must_use] attribute; flags all discarded return values.

6. **mutex_overuse_rust.json** -- `"pipeline": "mutex_overuse"`. Match Mutex type usages: `(type_identifier) @ty` filtered in post-processing or use `(generic_type type: (type_identifier) @outer)` to find Mutex<T>. Simplified: match `(call_expression function: (scoped_identifier) @fn)` for Mutex::new calls. Pattern: "mutex_overuse", severity: "info".

7. **pub_field_leakage_rust.json** -- `"pipeline": "pub_field_leakage"`. Use match_pattern to find struct fields with pub visibility: `(field_declaration (visibility_modifier) @vis name: (field_identifier) @name)`. Pattern: "pub_field_leakage", severity: "warning". Document: detects pub fields on structs; does not check if struct itself is pub.

8. **missing_trait_abstraction_rust.json** -- `"pipeline": "missing_trait_abstraction"`. Use select: "symbol" with where: {kind: ["function"], exported: true}. Pattern: "missing_trait_abstraction", severity: "info". Document: simplified; flags exported functions that could be trait methods.

9. **async_blocking_rust.json** -- `"pipeline": "async_blocking"`. Match std::fs, std::thread::sleep, std::net calls inside async functions. Use `(call_expression function: (scoped_identifier) @fn)`. Pattern: "blocking_in_async", severity: "warning". Document: flags all scoped calls, cannot verify async context.

10. **magic_numbers_rust.json** -- `"pipeline": "magic_numbers"`. Match numeric literals: `(integer_literal) @lit`. Pattern: "magic_number", severity: "info". Document: simplified; does not exclude const/static contexts or common values (0, 1).

11. **dead_code_rust.json** -- `"pipeline": "dead_code"`. Use select: "symbol" with where: {exported: true}, exclude: {is_test_file: true}. Pattern: "potentially_dead_export", severity: "info". Document: simplified from Rust; no cross-file reference counting. Flags all exported symbols as candidates.

12. **duplicate_code_rust.json** -- `"pipeline": "duplicate_code"`. Use select: "symbol" with where: {kind: ["function", "method"]}, then flag functions as candidates. Pattern: "potential_duplication", severity: "info". Document: "Simplified from Rust: rolling-hash body comparison not expressible in JSON. Flags function-level structural indicators only."

13. **coupling_rust.json** -- `"pipeline": "coupling"`. Use match_pattern to count import statements: `(use_declaration) @import`. Pattern: "high_coupling", severity: "info". Document: simplified; counts use declarations per file as coupling proxy. Does not compute cross-file dependency fan-out.
