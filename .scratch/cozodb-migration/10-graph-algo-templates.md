# 10 — DEPRECATED: graph-algo built-ins unavailable upstream

**Type:** N/A
**Label:** wontfix

## Status

**Deprecated.** Blocked on an unresolvable upstream dependency issue:
the `cozo-ce` crate's `graph-algo` feature transitively pulls
`graph_builder 0.4.1`, which uses an outdated rayon API (`into_par_iter`
returning a borrowing iterator) that doesn't compile against rayon
≥ 1.8. virgil-cli targets rayon 1.11. `graph_builder` has had no
release since 2023; no upstream fix is in flight.

The same blocker prevented us from enabling `graph-algo` in issue 02.

## What we couldn't ship

The original plan called for swapping hand-rolled recursive Datalog in
issue 05's templates for Cozo's parallel built-ins, plus 7 new templates:

| Template | Built-in it would have used |
|---|---|
| `find_cycles` swap | `ConnectedComponents(strong: true)` |
| `import_depth` swap | `TopSort` + max-aggregation |
| `find_callers` swap | `BFS` (backward) |
| `find_callees` swap | `BFS` (forward) |
| `shortest_call_chain` | `ShortestPathBFS` |
| `k_alternative_paths` | `KShortestPathYen` |
| `function_pagerank` | `PageRank` |
| `bridge_functions` | `BetweennessCentrality` |
| `module_clusters` | `CommunityDetectionLouvain` |
| `import_pagerank` | `PageRank` over `edge_imports` |
| `unreachable_from_main` | `BFS` + negation |

## Triage Notes

> *This was generated by AI during triage.*

Three ways someone could resurrect this issue later:

1. **Wait for cozo-ce upstream to fix the `graph_builder` dep.** Either a
   newer `graph_builder` release that's rayon-current, or cozo-ce
   migrating to a different parallel-graph crate.
2. **`[patch.crates-io]` fork of `graph_builder`.** One-line fix to its
   `into_par_iter()` call site. Possible but adds maintenance burden —
   anyone bumping rayon downstream of us would need to keep the patch in
   sync.
3. **Implement the 7 templates as Rust-side handlers** using a
   stand-alone graph-algo crate (`petgraph` was deleted in issue 09, but
   crates like `rustworkx-core` exist). The trade-off: Rust code per
   algorithm instead of Cozoscript. Each template becomes a `lookup()`
   entry in `src/queries/rust_templates.rs` that queries `*symbol` /
   `*edge_calls` / `*edge_imports`, builds an adjacency list, runs the
   algorithm, and emits audit-shape rows.

The original templates in issue 05 (recursive-Datalog versions of
`find_cycles`, `import_depth`, `find_callers`, `find_callees`) continue
to work — they're slower than the built-in equivalents would be, but
they ship and are good enough for typical workspace sizes.

## Recommendation

Leave deprecated. Re-evaluate if/when cozo-ce ships a working
`graph-algo` against current rayon, or if there's a concrete user need
for one of the missing analyses (PageRank, Louvain, etc.) that
justifies the Rust-side handler path.
