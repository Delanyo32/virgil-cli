# 10 — Adopt Cozo `graph-algo` built-ins + 7 new templates

**Type:** AFK
**Label:** ready-for-agent

## Parent

`.planning/proposals/cozodb-migration.md` — Phase 10.

## What to build

Swap hand-rolled recursive Datalog in Phase-5 templates over to Cozo's
parallel `graph-algo` built-ins, and ship 7 new templates that expose
analyses the legacy DSL couldn't express.

**Swap-ins (rewrite existing templates):**

- `find_cycles.cozoql` → `ConnectedComponents(strong: true)`
- `import_depth.cozoql` → `TopSort` + max-aggregation
- `find_callers.cozoql` → `BFS` (direction: backward)
- `find_callees.cozoql` → `BFS` (direction: forward)

**New templates:**

- `shortest_call_chain.cozoql` — `ShortestPathBFS`
- `k_alternative_paths.cozoql` — `KShortestPathYen`
- `function_pagerank.cozoql` — `PageRank` over call graph
- `bridge_functions.cozoql` — `BetweennessCentrality`
- `module_clusters.cozoql` — `CommunityDetectionLouvain`
- `import_pagerank.cozoql` — `PageRank` over `edge_imports`
- `unreachable_from_main.cozoql` — `BFS` + negation against the result set

Each new template needs 1–2 worked examples in its file header so users
can adapt it.

No new CLI flags — all of these are `--template <name> --param k=v`.

## Triage Notes

> *This was generated by AI during triage.*

`PageRank`, `Louvain`, and `BetweennessCentrality` all take tuning
parameters (damping factor, resolution, normalization). Pick documented
defaults for each, expose them as `--param` overrides, and call out the
defaults in the template's header comments so users know what they're
getting before they tune.
