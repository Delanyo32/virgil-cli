# 03 — CfgFactBuilder + 9 language CFG builders emit Cozo facts

**Type:** AFK
**Label:** ready-for-agent

## Parent

`.planning/proposals/cozodb-migration.md` — Phase 3.

## What to build

Replace the in-memory `FunctionCfg`/`BasicBlock` graph construction with a
write-only `CfgFactBuilder` accumulator that each language CFG builder
emits into. The absorber drains accumulators and writes the CFG relations
to Cozo.

Scope:

- New `src/graph/cfg/facts.rs` — `CfgFactBuilder` (write-only accumulator).
- New `src/graph/cfg/emit.rs` — drains the builder into Cozo rows.
- Rewrite every language CFG builder under `src/languages/<lang>/cfg.rs`
  (TS, JS, C, C++, C#, Rust, Python, Go, Java, PHP) to emit through the
  fact builder instead of constructing `petgraph::DiGraph<BasicBlock,
  CfgEdge>` values. The trait in `src/languages/cfg.rs` shifts from
  "return a `FunctionCfg`" to "write into a `CfgFactBuilder`".
- All per-function CFG relations declared in Phase 2's schema get populated:
  `cfg_block`, `cfg_edge`, `cfg_assign`, `cfg_call`, `cfg_return`,
  `cfg_guard`, `cfg_resource_acquire`, `cfg_resource_release`, `cfg_phi`.
- Per-language fact-shape tests: small fixture per language → assert
  expected rows exist for representative constructs (if/while/try/return).

`CodeGraph::cfg_for_function` and the existing lazy cache continue to work
during this phase (they remain the source for taint/resource analysis
until those subsystems are deleted in Phase 6).
