# 03 — DEPRECATED: CFG facts won't be stored

**Type:** N/A
**Label:** wontfix

## Status

**Deprecated** — same reasoning as issue 04's metric-facts deprecation.
CFG is recomputable cheaply on demand via `CodeGraph::cfg_for_function`
(which already exists and caches lazily). Materialising 9 detailed CFG
relations into Cozo would duplicate the source-of-truth, force
stale-fact invalidation work in issue 08, and only buys "pure
Cozoscript" templates for two analyses (taint, resource) that work fine
as Rust-side handlers.

## Triage Notes

> *This was generated by AI during triage.*

The deprecation cascades to two other issues — neither blocked by this
file any more:

- **Issue 05** — `taint_paths.cozoql` and `unreleased_resources.cozoql`
  cannot be pure Cozoscript. They join the small club of Rust-side
  `--template` handlers alongside `complexity_hotspots`. The handler
  pattern is: parse params, call into `src/graph/taint/` or
  `src/graph/resource.rs`, format results in the audit-shape tuple.

- **Issue 06** — the deletion list shrinks. `src/graph/taint/`,
  `src/graph/resource.rs`, `src/graph/cfg.rs`, and the `cfg_for_function`
  / `cfg_cache` / `inject_cfg` / `function_cfg_indices` machinery on
  `CodeGraph` all **stay**. Only the JSON DSL stack (`src/pipeline/`,
  `src/audit/`, audit subcommand, JSON pipeline parsing in `projects
  query`) goes away.

The schema in `src/cozo/schema.rs` was already written to **not**
include any `cfg_*` relations (issue 02 only declared the cross-function
graph). No code rollback needed — issue 02 + 04 form the entirety of
the Cozo facts that get materialised.

## Original scope (for reference)

The proposal's Phase 3 specified:
- `src/graph/cfg/facts.rs` `CfgFactBuilder`
- `src/graph/cfg/emit.rs`
- Rewrites of 9 language CFG builders
- 9 `cfg_*` Cozo relations

None of that will be built.

## What replaces it

Nothing — `CodeGraph::cfg_for_function` already does the job. Future
templates that need CFG access call into Rust, the same way
`compute_metric` is called today.
