## What to build

Add the user-facing Cozoscript query surface to the `projects query`
subcommand and ship the first 10 built-in templates.

CLI surface (additive — existing `--q '{json}'` keeps working until
Phase 6 deletes it):

- `--cozoscript '<inline>'` — power-user mode
- `--file <path.cozoql>` — load script from file
- `--template <name>` — invoke a built-in template
- `--param k=v` — parameter binding (repeatable). Parameters bound as
  `$param` in Cozoscript; never interpolated into the script text
- `--read <file>` and `--lines` continue to work unchanged

Output convention: a query returning columns
`(file, line, severity, pattern, message)` (plus any extras) is
auto-formatted as audit findings. Other shapes use existing
`--format json|csv|table`.

Ship these 10 templates in `src/queries/builtin/`:

- `find_callers.cozoql`
- `find_callees.cozoql`
- `find_cycles.cozoql`
- `find_function_by_name.cozoql`
- `complexity_hotspots.cozoql`
- `export_surface.cozoql`
- `taint_paths.cozoql`
- `unreleased_resources.cozoql`
- `import_depth.cozoql`
- `unused_symbols.cozoql`

The proposal includes worked Cozoscript for `find_callers`, `find_cycles`,
and `complexity_hotspots` — use those verbatim as starting points. The
remaining 7 follow the same shape.

## Triage Notes

> *This was generated by AI during triage.*

The proposal includes worked Cozoscript for only 3 of the 10 templates
(`find_callers`, `find_cycles`, `complexity_hotspots`). The other 7 must
be authored from scratch. Subtle Datalog errors can pass a small fixture
test but miss real findings, so each template needs a real-corpus smoke
test (counts of rows returned on a representative project), and the
results should be posted in the PR description for review.

**Three templates need Rust escape hatches**, not pure Cozoscript:

- **`complexity_hotspots`** — metric facts were deprecated in issue 04.
  The handler queries `?[id, name, file_path, start_line, end_line] :=
  *symbol{...}` filtered by `*file_classification{is_test: false}`,
  calls `crate::pipeline::stages::compute_metric::*` to compute
  cyclomatic + length on demand, then emits the audit-shape tuple.

- **`taint_paths`** — CFG facts were deprecated in issue 03. The handler
  resolves source/sink patterns to symbols via Cozoscript, then calls
  `src/graph/taint/` for the actual intra-function flow analysis, and
  emits results as audit-shape tuples.

- **`unreleased_resources`** — same reason as `taint_paths`. Handler
  calls `src/graph/resource.rs` for the lifecycle check, formats
  findings.

The remaining 7 templates (`find_callers`, `find_callees`, `find_cycles`,
`find_function_by_name`, `export_surface`, `import_depth`,
`unused_symbols`) are pure Cozoscript over the cross-function graph.

## Acceptance criteria

- [ ] `--cozoscript`, `--file`, `--template`, `--param` flags work and are
      mutually exclusive with each other (and with `--q`/`--read`)
- [ ] Template loader discovers all 10 builtins
- [ ] Parameter binding uses `$param`; injection-test with adversarial
      `--param` values cannot escape into the script body
- [ ] Each of the 10 templates runs on a fixture corpus and returns the
      expected rows

      workspace (e.g. the virgil-cli source tree itself) and the row
      counts + first 5 rows are posted in the PR description
- [ ] Audit-shaped output (`file,line,severity,pattern,message`)
      auto-formats as findings; other shapes respect `--format`
- [ ] `cargo test` green
