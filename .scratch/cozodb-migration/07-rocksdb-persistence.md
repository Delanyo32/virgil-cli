**Type:** AFK

## What to build

Move the Cozo store from in-memory to on-disk so successive `projects query`
invocations don't re-parse the workspace.

Scope:

- Build writes the Cozo store to `~/.cache/virgil/<workspace_hash>.cozo/`.
  Hash key derived from the workspace identity (e.g. registered project
  name + absolute path, or S3 URI).
- Subsequent invocations open the existing store and skip the build pass
  if `build_meta_files` shows nothing changed (basic mtime/hash check —
  full incremental refresh is a separate slice).
- Decide RocksDB vs SQLite as the default backend. Benchmark both on a
  representative workspace (openclaw is the proposal's reference) and
  record the result in `build_meta` + a short note in the release notes.
- Multi-process safety: CLI build takes an exclusive write lock; `serve`
  opens read-only.
- Schema versioning: `build_meta.schema_version` mismatch → rebuild from
  scratch.

## Triage Notes

> *This was generated by AI during triage.*

The RocksDB-vs-SQLite default is a deferred decision in the proposal
(open question #2). Before locking the default in the PR, post the
benchmark numbers (cold build time, warm query time, on-disk size, peak
RSS) as a PR comment and wait for maintainer sign-off. Both backends
remain compiled in either way; the change is only which one is used by
default.
